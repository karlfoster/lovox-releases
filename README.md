# Lovox Releases

Public update feed and release downloads for [Lovox](https://github.com/karlfoster/Lovox) ŌĆö a macOS push-to-talk voice transcription app.

> **Why a separate repo?** The main Lovox repo is private (source code). Sparkle (the macOS update framework) needs publicly accessible URLs to check for updates and download DMGs. This repo provides that public surface without exposing source code.

## Repository layout

```
lovox-releases/
  appcast.xml     # Sparkle update feed ŌĆö lists all versions + download URLs
  index.html      # Minimal landing page (required by GitHub Pages)
  README.md
```

Release DMGs are attached as **assets on GitHub Releases** in this repo (not committed to the repo itself).

## Feed URL

The appcast is served via GitHub Pages at:

```
https://karlfoster.github.io/lovox-releases/appcast.xml
```

This URL is hardcoded in the Lovox app's `Info.plist` as `SUFeedURL` (Sparkle's feed key).

## How updates work

1. A new version of Lovox is built, signed, and notarized
2. The DMG is attached to a new GitHub Release on this repo (e.g. `v1.7.0`)
3. `appcast.xml` is updated with a new `<item>` entry containing the version, download URL, release notes, and EdDSA signature
4. Sparkle (embedded in the running Lovox app) periodically polls `appcast.xml`
5. If a newer version is found, the user is prompted to update
6. Sparkle downloads the DMG, extracts the `.app`, replaces the old version, and relaunches

## appcast.xml format

Each release is an `<item>` in the RSS channel:

```xml
<item>
  <title>Version X.Y.Z</title>
  <description><![CDATA[<p>Release notes here</p>]]></description>
  <pubDate>Mon, 24 Mar 2026 12:00:00 +0000</pubDate>
  <enclosure
    url="https://github.com/karlfoster/lovox-releases/releases/download/vX.Y.Z/Lovox-X.Y.Z.dmg"
    length="12345678"
    type="application/octet-stream"
    sparkle:version="X.Y.Z"
    sparkle:edSignature="BASE64_SIGNATURE_HERE" />
  <sparkle:minimumSystemVersion>14.0</sparkle:minimumSystemVersion>
</item>
```

- `sparkle:version` ŌĆö the build/marketing version (matches `CFBundleShortVersionString`)
- `sparkle:edSignature` ŌĆö EdDSA (Ed25519) signature for download integrity (generated by Sparkle's `sign_update` tool)
- `sparkle:minimumSystemVersion` ŌĆö minimum macOS version required

## Updating the appcast

The release script in the main Lovox repo handles this automatically. Manual steps if needed:

1. Add a new `<item>` to `appcast.xml` (newest first)
2. Commit and push ŌĆö GitHub Pages deploys automatically within ~60 seconds
